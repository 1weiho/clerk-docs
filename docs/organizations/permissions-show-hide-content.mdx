# Use custom permissions to show and hide content from users

## Outcomes

- use `<Gate />` on the frontend
- be able to use has in a custom flow to hide sections of content to users with different roles

## Note

You can update members ship both from the dashboard (admin/developer/admin of org view), and also from the organization profile (end user view).
